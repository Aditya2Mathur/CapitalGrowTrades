# Capital Grow Trades - Landing Page

A modern, single-page landing website for a Telegram Trading Community. Designed with a dark mode aesthetic, neon glowing accents, and a dynamic "anti-gravity" interactive UI.

## Features
- **Dark Mode Aesthetic**: Deep dark backgrounds with neon green (`#00ff88`) and neon blue (`#00d2ff`) glowing highlights.
- **Google Anti-Gravity Effect**: UI elements (floating charts, coins, badges) gently drift and parallax as the cursor moves across the screen using Linear Interpolation (Lerp).
- **Trader-Focused UI**: Custom "Trade Example" card styled like a professional trading dashboard component to showcase Entry, Target, and Stop Loss details along with a pulsing active signal dot.
- **Responsive Design**: Fully responsive layout optimized for mobile, tablet, and desktop viewing.
- **Telegram Integration**: Prominent Call-To-Action buttons directing users to the official Telegram channel.
- **Glassmorphism Panels**: Frosted glass effects on cards and floating elements.

## Tech Stack
- **HTML5**: Semantic markup and structure
- **CSS3 (Vanilla)**: Custom styling, CSS Variables, Flexbox/Grid layouts, Animations
- **JavaScript (Vanilla)**: Mouse tracking, Intersection Observers for scroll reveals, Lerp mathematical calculations for the anti-gravity effect.
- **FontAwesome**: Scalable vector icons for the UI.
- **Google Fonts**: 'Outfit' (Headings) and 'Inter' (Body).

## Project Structure
- `index.html`: The main landing page document.
- `style.css`: Contains all styles including layouts, themes, responsive queries, and animations.
- `script.js`: Contains the logic for the Google Anti-Gravity floating effect, smooth scrolling, and scroll reveals.
- `logo.jpeg`: The brand logo for Capital Grow Trades.

## How to Run Locally
1. Clone or download this repository.
2. Open the `index.html` file in any modern web browser.
*(Alternatively, for the best experience, run a local development server such as VS Code Live Server or Python's `http.server` to avoid any CORS issues with local file loading, though this project does not strictly require it).*
